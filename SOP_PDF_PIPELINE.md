# 任務名稱：PDF 深度結構化與語意增強自動化產線

## 【執行目標】

自動化處理指定目錄下的 PDF 文件，將其轉化為具備「圖表文字化」與「語意摘要」的高價值 Markdown 格式。

## 【工作流配置步驟】

### 第一階段：結構化解析 (Parsing)

1. 調用解析模組（如 LlamaParse），將 PDF 原始內容轉換為 Markdown。
2. 執行「版面分析」，識別標題層級（H1, H2, H3）並保留表格的結構。
3. 自動提取文中所有圖片與圖表，將其儲存至臨時資料夾，並在原位保留圖片預留位 [IMAGE_REF: ID]。

### 第二階段：視覺轉譯 (VLM Analysis)

1. 遍歷臨時資料夾中的所有圖表圖片。
2. 調用多模態模型（如 Gemini / GPT-4o），執行 Prompt：「請精準描述此圖表的數據、趨勢與核心結論，並以純文字呈現，不使用標題。」
3. 將返回的描述文字，自動插回原 Markdown 的 [IMAGE_REF: ID] 標記處。

### 第三階段：語意增強 (Enrichment)

1. 根據 Markdown 標題層級進行切塊 (Chunking)。
2. 針對每一章節區塊，調用 LLM 生成以下 Metadata：
   - [摘要]：50 字以內的內容概述。
   - [關鍵字]：3-5 個核心術語。
   - [核心概念]：本段欲傳達的最重要知識點。
3. 將 Metadata 以 YAML Frontmatter 格式插入該段落的最上方。

## 【輸出要求】

- 格式：標準 Markdown (.md)
- 語系：繁體中文
- 存檔：輸出至 /Volumes/2T/Structured_Data 資料夾

## 【觸發機制】

未來接收到 PDF 檔案時，主動詢問用戶：「是否執行 PDF 深度結構化產線？」
